Historique

AngularJS est un framework Javascript SPA (Single Page Application) opensource depuis existant depuis 2009

Développé par initialement par Miško Hevery et Adam Abronsw puis reprit par Google.

Actuellement en version 1.5, la version 1.0.0 n'est arrivée qu'en juin 2012

Il propose d'étendre le HTML en y ajoutant des nouvelles fonctionnalités et ou balises.

Il suit le pattern MVC

Il offre le binding bidirectionnel

Il utilise l'injection de dépendance

Il utilise JQlite en remplacement de JQuery mais peut aussi utiliser JQuery si il est chargé

Il a déjà son successeur avec AngularIO ou Angular 2.0

Le pattern MVC 1/2

L'idée est de bien séparer les données, la présentation et les traitements

M Le modèle Le coeur de l'application, contient l'intelligence métier et/ou les données. Dans notre cas les services.
V La Vue C'est la partie visible et avec laquelle va interagir l'utilisateur final. Dans notre cas le HTML et le CSS. La vue n'éffectue pas de traitement directement et passe par le controlleur.
C Le Controlleur Il s'occupe de gérer la communication entre la vue et le modèle. Il ne doit pas faire de traitement en s'appuyant sur le modèle. Si une interaction necessite un changement de donnée demande ce changement au modèle

Le pattern MVC 2/2

Avantage

Il permet d'apporter de la clarté dans l'architecture et simplifie le travail des developpeurs. La vue peut etre concu ou évoluer indépendament du modèle et vice et versa.

Exemple

  • un utilisateur interagit avec l'application, l'action est analysée puis transmise au contrôleur (saisie dans un champs texte)
  • le contrôleur demande au modèle demande le traitement et notifie à la vue que l'action est traitée (par exemple convertie en kebab-case)
  • la vue notifiée fait une requête au modèle pour se mettre à jour (par exemple affiche la valeur saisie en kebab-case dans une zone de texte).

Dans angular

AngularJS simplifie l'implémentation du pattern MVC par l'implementation de Two-Way data-binding.

La notion de data-binding

Ce concepte permet de lier la vue au modèle. L'implementation dans AngularJS est bi-direcetionnel donc on parle de Two-Way data-binding.

Donc dans ce cas un changement dans la vue peut impacter la donnée dans le modèle et la mise à jour coté modèle sera automatiquement impacté dans la vue

L'injection de dépendances

L'injection de dépendances permet à une fonctionnalité de ne pas se soucier de l'instanciation des dépendances dont elle a besoins.

Avantage:

  • Homogeneisation des dependances pour pouvoir etre injectable
  • Possibilité de changer d'implémentation, pour les tests par exemple

Les services

Les services sont des briques fonctionnelles directement utilisable dans l'application via le systeme d'injection de dépendance d'Angular.

Important : Se sont des singletons.

De multiple services sont a disposition dans Angular tel que :

  • $http : pour interagir avec la couche reseau fair des requetes
  • $cookies : pour interagir avec les cookies
  • $location: pour interagir avec la barre d'adresse du navigateur

$scope et $watch

le $scope

Le $scope est une dépendance accessible dans la vue et dans le controlleur

Toutes les données ou fonctionnalités doivent être rattaché au $scope pour pouvoir être éxposé à la vue

Une application AngularJS possède un ou plusieurs $scope et un seul $rootScope qui permet d'accéder à l'ensemble des contextes présents dans votre page.

le $watch

Le $watch permet de definir des actions sur l'observation de la modification d'une donnée ou du résultat d'une fonction

Le $watch est un outil important, puissant mais à utiliser avec partiemonie car il peut être tres consommateur en ressource

Global State
